# Sermonator ‚Äî Bundle 2: Deep-Compat (Design)

- **Date:** 2026-06-27
- **Status:** DESIGN. Gated design workflow (proposals ‚Üí adversarial critique ‚Üí synthesis). The FINAL parity-roadmap bundle (Tier B deep-compat).
- **Sizing note (¬ß63 guard):** the precursor migrator-reality audit is UNMET (no real Pro-site sample), so Bundle 2 stays **full-scope** by default. Only page-builder is genuinely blind; the roadmap defines its full-scope deliverable as **detection + warning** (rebuild already won't-do), so detection-only is full-scope here, not a collapse. Every deferral is recorded as an **explicit signed Contract-guard exception** in the PR ‚Äî never a silent down-size.

## 1. Scope
1. **Attribute-faithful `[sermons]`/`[sermons_sm]`** ‚Äî replace the Bundle 1 safe-default+notice shim with a per-attribute ledger mapping legacy attributes onto the existing `SermonQuery`/`GridArgs`/`Renderer` engine; drop the review notice per faithfully-reproducible attribute, keep a **precise per-attribute** notice for anything unvalidatable against the absent SM source.
2. **Per-podcast feed filtering** ‚Äî a read-only scope resolver reading the already-migrated `META_PODCAST_SETTINGS` term-scope keys ‚Üí `SermonQuery` `tax_query`; empty scope = today's exact unscoped query (single-podcast provably unchanged); a **never-serve-empty** invariant on unresolved scope.
3. **Page-builder detection** ‚Äî a read-only, fail-visible scanner in the migration report + Site Health; the module rebuild is explicitly backlogged.

## 2. Key decisions (critique-hardened)
1. **Per-attribute ledger** (not blanket-notice, not all-faithful) ‚Äî the Contract's binding rule is literally per-attribute. A `LegacyAttributeMapper` classifies each attr FAITHFUL / NO-OP-SAFE / UNVALIDATABLE / UNSUPPORTED, applies the faithful ones, and names ONLY the present unfaithful ones in the notice (empty ‚Üí no notice).
2. **`per_page` needs REAL pagination.** The native grid renders a fixed count with no pager, so a naive `per_page` map silently drops the long tail of a large archive (the canonical fail-wrong). Build a paginated list variant + escaped pager on a **dedicated `sermon_page` query var that MUST be registered** via the `query_vars` filter (WP `get_query_var` returns empty for any unregistered var ‚Üí the pager would be stuck on page 1 ‚Üí silent tail-drop). **Read-path is pinned:** the embedded current-page is read from `sermon_page`, **NOT** `GridArgs::currentPage()` (which reads the MAIN query's `paged`/`page` ‚Äî those stay reserved for the sermon ARCHIVE; on a static embedding page `paged` is 1, so reusing `currentPage()` would make pages 2..N unreachable). Pager links are built with `add_query_arg('sermon_page', N)` on the current request URL (query-string form), explicitly **NOT** pretty `/page/N/` permalinks (which collide with the main query and 404 on a static page ‚Äî reconciling `Renderer::grid()`'s documented constraint). Until the pager lands, `per_page` is NOT certified faithful (named in the notice + fires `sermonator_list_truncated`). The certifying integration test MUST embed `[sermons per_page=N]` on a **NON-archive (static) page** and prove page 2 loses no content.
3. **Preached-date semantics diverge from native ‚Äî add a `dateScope` mode to `SermonQuery`** (`inclusive`=native default LEFT-JOIN dateless-last+future; `preached`=EXISTS + `META_DATE <= now` dropping future AND dateless, matching legacy `display_sermons()`; `none`=no date meta_query). NEVER fork/mutate the native grid's OR/NOT-EXISTS branch. `META_DATE` compared as NUMERIC (signed, so pre-1970 negatives sort correctly). **`order`/`orderby` defaults + `orderby=date` resolution are OPTION-driven, exactly as SM** (NOT hardcoded): a bare `[sermons]` takes its default `order` from the migrated `Identifiers::OPTION_ARCHIVE_ORDER` (fallback `desc`) and default `orderby` from the migrated `Identifiers::OPTION_ARCHIVE_ORDERBY` (fallback `date_preached`‚Üí`dateScope=preached`) ‚Äî both preserved by `OptionWriter`'s wholesale `sermonmanager_*`‚Üí`sermonator_*` copy (`display_sermons()` :788-789). `orderby=date` is **NOT** mapped to published unconditionally; it resolves to published date ONLY when the migrated `OPTION_ARCHIVE_ORDERBY === 'date'`, else to `dateScope=preached` ‚Äî byte-identical to `display_sermons()` :866. **The date filters `year`/`month`/`before`/`after` are gated on `dateScope=preached`** ‚Äî SM applies them ONLY inside the `meta_value_num` branch (`:896`, `:940`); under any non-preached orderby they are bug-compatible no-ops, matching SM (otherwise `[sermons orderby="title" year="2020"]` would over-filter and render a SMALLER set than SM).
4. **`after` is UNVALIDATABLE ‚Äî bug-for-bug exact-equality (corrected rationale).** SM's `=>` is NOT a valid `WP_Meta_Query` operator; WP normalizes the unknown operator to `=`, so SM's clause became `META_DATE = strtotime(after)` (ANDed with `META_DATE <= now`, gated on `dateScope=preached`, `display_sermons()` :957) ‚Äî an exact-day, near-EMPTY match, **not** an ignore. Silently ignoring `after` would render the FULL list (content SM never produced = fail-wrong). We best-effort reproduce SM's exact-equality (`META_DATE = strtotime(after)`, NUMERIC/signed, gated on `dateScope=preached`) AND **KEEP a precise per-attribute notice** when `after` is present ‚Äî the `strtotime`/timezone reconstruction cannot be validated against the absent running SM, so it stays fail-visible.
5. **Numeric legacy ids: resolve-or-drop, never pass-through.** `Crosswalk::LEGACY_TERM_ID`/`LEGACY_POST_ID` are strippable back-refs (resolvable pre-Finalize, gone post-Finalize). Resolve `include`/`exclude`/numeric `filter_value` via the crosswalk; on non-resolution DROP that axis + name it. Passing legacy id 42 through would select a DIFFERENT new term/post 42 (fail-wrong). Prefer slug resolution (durable across Finalize). **Numeric `filter_value` KEEPS its notice even on clean crosswalk resolution:** SM's numeric branch computed `$field='id'` but ALWAYS built the `tax_query` with `'field' => 'slug'` (`display_sermons()` :1040) ‚Äî a numeric `filter_value` matched a term whose SLUG equals the number, virtually always EMPTY. Rendering the crosswalk-resolved term's sermons is a faithful reinterpretation of *intent*, not a reproduction of SM's near-empty output ‚Üí it stays UNVALIDATABLE and notice-kept (only `include`/`exclude` and slug `filter_value` earn the notice-off).
6. **One shared `LegacyTermResolver`** consumed by BOTH the `[sermons]` mapper AND the feed scope path (slug‚Üínew via durable taxonomy slug; numeric‚Üínew via `TermCrosswalk`; reports unresolved) ‚Äî two divergent resolvers would give two different answers to "what survives Finalize."
7. **Per-podcast scope source = `META_PODCAST_SETTINGS` keys intersected with `ID::sermonTaxonomies()`** (NOT a hardcoded five-slug list ‚Äî mirrors `PodcastMetaSchema::keys()` anti-drift). Values are already NEW term ids (remapped by `PodcastWriter::remapSettingsTerms()`), fed to `SermonQuery::buildTaxQuery` (relation=AND across taxonomies, IN within ‚Äî byte-identical to Pro's `filter_the_query`). No new tax-query builder.
8. **Feed fail-visible + never-serve-empty:** clean resolve ‚Üí apply, no notice (earned). An open `missing_podcast_term_crosswalk:*` flag (Pro had scope but a term didn't resolve) ‚Üí fall back to UNSCOPED + fire `sermonator_feed_scope_incomplete`; **never serve a feed scoped to a dead term id** (would silently empty a live feed). `>1` published podcast with no scope keys ‚Üí unscoped + the existing over-inclusion signal. Empty scope on a single podcast ‚Üí today's exact query, silent.
9. **Live feed item-set change is irreversible** (rollback story 8.1) ‚Äî per-podcast filtering narrows the subscriber-visible set; before any church switches, a per-podcast pre/post feed-diff via the existing `LegacyFeedSnapshot` must pass. GUID continuity protects *surviving* items only.
10. **Audio/video `sermons_to_show` modes DEFER** ‚Äî audio-only ships; `video`/`*_priority` fire `sermonator_feed_mode_unsupported` and the per-podcast notice is NOT retired while mode faithfulness is unbuilt.
11. **Page builders = read-only fail-visible scanner (Option B), rebuild backlogged.** `PageBuilderScanner` fingerprint FLOOR (required catch-all): (a known builder meta key present ‚Äî `_elementor_data`, Divi `_et_pb*`, Beaver `_fl_builder_data`, WPBakery `vc_*`) AND (any legacy sermon reference ‚Äî a `wpfc_sermon` id, a legacy taxonomy slug, or a legacy `[sermons]`/`[list_sermons]`/`[latest_series]`/`[sermon_images]` string). A distinct lower-severity finding for legacy shortcodes embedded in builder postmeta (which the `do_shortcode` shim does NOT fire); do NOT build a do_shortcode-on-meta bridge. Surface in BOTH the migration report (pre-switch) AND Site Health.
12. **¬ß63 prevalence counter** ‚Äî emit detect/verify counts (podcasts-with-scope, `>1`-podcast sites, single-scoped-podcast count, embedded-`[sermons]` attribute density, builder findings) so the first real migration produces the data the audit lacked; future video-mode + object-term-mirroring work is gated on it.

## 3. Components
`Frontend\Compat\LegacyAttributeMapper` (raw-atts ‚Üí `{gridArgs,dateScope,orderby,postIn,postNotIn,unfaithfulAttrs}`) ¬∑ `Frontend\Compat\LegacyTermResolver` (shared slug/numeric term resolver) ¬∑ `Frontend\Compat\LegacyPostResolver` (legacy post id ‚Üí new) ¬∑ `Frontend\SermonQuery` (+`orderby`, `dateScope` mode, NUMERIC date-range bounds, `post__in`/`post__not_in`) ¬∑ `Frontend\Renderer` (+paginated list variant + escaped pager reading the **registered** `sermon_page` query var ‚Äî NOT `GridArgs::currentPage()`'s `paged`/`page` ‚Äî with `add_query_arg` query-string links, never pretty `/page/N/`) ¬∑ `Frontend\Compat\LegacyShortcodes` (rewritten `[sermons]`/`[sermons_sm]` ‚Üí mapper‚ÜíSermonQuery‚ÜíRenderer + precise notice) ¬∑ `Frontend\Feed\PodcastScopeResolver` (`forPodcast(id)` ‚Üí `array<taxonomy,list<int>>`) ¬∑ `Frontend\Feed\PodcastFeed` (+`items(scope)` + never-serve-empty fallback + fail-visible signals) ¬∑ `Migration\PageBuilderScanner` (read-only detector ‚Üí report + Site Health) ¬∑ migration detect/verify prevalence counter ¬∑ the Compatibility Contract (per-attribute ledger + scope rules + builder rows + ¬ß63 exceptions ‚Äî PR exit criterion).

## 4. Top risks
1. **Silent `per_page` under-render** ‚Üí real pagination + `sermonator_list_truncated` until certified.
2. **Reconstructed preached/date semantics have no running-SM oracle** ‚Üí certify faithful only behind pinned integration tests; everything else keeps a notice.
3. **Post-Finalize numeric-id fail-wrong** ‚Üí resolve-or-drop, never pass-through.
4. **Editor-only notices invisible to visitors/podcast apps** ‚Üí every visitor-facing fail-open ALSO fires an observable `do_action`.
5. **Live per-podcast feed narrowing is irreversible** ‚Üí pre/post feed-diff gate before switch.
6. **Migration-fidelity gap** ‚Äî Pro variants storing scope only in podcast object-terms migrate as empty scope (recorded; the prevalence counter measures it).
7. **Undetected page builder defeats fail-visible** ‚Üí the coarse catch-all floor is required, not optional.
8. **¬ß63 scope-collapse regret** ‚Üí every deferral a signed Contract exception, not silent.

## 5. Test strategy
Unit (Brain Monkey): the mapper ledger (one test per cell ‚Äî `orderby=date` resolved against the migrated `OPTION_ARCHIVE_ORDERBY` both ways, option-driven default `order`/`orderby`, `after` bug-for-bug exact-equality + notice-kept, `year`/`month`/`before`/`after` gated on `dateScope=preached` and no-op under a non-preached orderby, `filter_by` slug crosswalk, numeric `filter_value` notice-kept-on-resolution, unknown-attr capture, numeric resolve/drop); `LegacyTermResolver`/`LegacyPostResolver` pre/post-Finalize transition; `PodcastScopeResolver` (scope read, missing-crosswalk ‚Üí unscoped, empty ‚Üí unscoped); `SermonQuery` dateScope modes; the scanner fingerprint + zero-writes. Integration (wp-env): native grid (dateScope=inclusive) byte-unchanged; dated/dateless/future/pre-1970 ordering; **`[sermons per_page=N]` embedded on a NON-archive (static) page reaches page 2 (via the registered `sermon_page` var) with no content lost** (the archive's `paged` stays 1); `OptionWriter` migrates `archive_orderby`/`archive_order` and the bare `[sermons]` honors them; per-podcast scoping serves only scoped sermons + the **HARD no-regression-on-single-podcast feed-diff** against `LegacyFeedSnapshot`; the scanner surfaces in report + Site Health. Update the Contract changelog (PR exit criterion).
