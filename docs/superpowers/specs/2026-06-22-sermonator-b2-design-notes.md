# Sermonator — Plan B2 Design Notes (adversarially verified)

- **Date:** 2026-06-22
- **Status:** Approved design (autonomous run). Produced by a parallel subsystem-design + 3-lens adversarial critique (data-loss / reversibility / idempotency-resume) + synthesis workflow. 37 holes surfaced, 12 critical — all closed by the task plan below.
- **Scope:** The migration *execution* engine (the write/verify/reverse half) for sub-project 1, on top of B1's read-and-transform core. Split for execution into **B2a (writers & crosswalk, tasks 1–19)** and **B2b (lifecycle: orchestrator/verifier/rollback/finalizer/CLI/e2e, tasks 20–25)**.

## Invariants (review these as single cross-cutting properties)

1. **Crash-safety spine (idempotency):** for every record, write the crosswalk back-ref **FIRST** (immediately after `wp_insert_*`, or via `meta_input` in the same call), and write a separate `MIGRATION_COMPLETE` flag **LAST**. The idempotency gate distinguishes *complete* from *stamped-but-partial* (resume/redo a partial; never skip-forever). Closes the duplicate-orphan, un-rollback-able-residue, and resume-correctness critical holes together.
2. **Non-destructive until Finalize:** nothing writes/alters/deletes legacy `wpfc_*`/`sermonmanager_*` data before an explicit Finalize. Every writer reads legacy read-only; every integration test snapshots legacy rows before/after and asserts byte-equality. Shared attachment posts are **never** mutated (referenced by id only).
3. **Ordering is enforced, not hoped:** Orchestrator hard-gates terms → artwork → podcasts/options → sermons. Term reconciliation is re-appliable on already-created posts so out-of-order/partial term runs self-heal (never let "post exists" short-circuit term repair).
4. **add_option-first + pre-migration backup:** before writing any `sermonator_*` option/term-images, back up a pre-existing native value to `OPTION_PRE_MIGRATION_BACKUP` so native config is never clobbered and rollback can restore it.
5. **Serialized/encoding correctness:** write meta from the per-key **unserialized** `get_post_meta($id,$key,false)` form (core re-serializes); never feed the no-key raw-serialized values to `add_post_meta`. `wp_slash` the post array. Disable KSES for inserts. Encoding-harden the checksum (`JSON_INVALID_UTF8_SUBSTITUTE` or `serialize()`).
6. **Three irreversible/destructive operations** (Orchestrator phase-commit, Rollback, Finalizer): Finalize is the only point of no return — gated on `state=='verified'` **AND** a fresh drift rescan, deleting per-record only for verified counterparts, stripping a strict back-ref allowlist that never touches preserved content (`LEGACY_POST_CONTENT`, unresolved `MIGRATION_FLAGS`).

## Gating criteria (carried from B1's final review) — how each is closed

- **(1) `post_content_temp` into the reconciler safety-net+flag:** Task 3 grows `reconcile()` to a 3-arg form running both `post_content` AND `post_content_temp` through a shortcode/HTML-aware safety-net (body-only-in-temp caught); Task 14 keeps the verbatim `post_content_temp` meta row as its single canonical home (reconciler only backs up text absent from both description and that row — no double-flag).
- **(2) normalized timestamp alongside raw, for non-numeric dates:** Task 2 timezone-anchored `DateNormalizer`; Task 15 writes `sermonator_date_normalized` for **every** non-numeric `sermon_date` row alongside the untouched verbatim raw (raw is source of truth; normalized is advisory + flagged).
- **(3) shared legacy-fixity checksum:** Task 6 extracts the exact Detector hash into `LegacyChecksum::forPost` (encoding-hardened); Detector delegates; Task 21 Verifier uses it ONLY as a legacy-fixity/drift oracle (source edited between detect and migrate) and proves target completeness via counts + legacy→target field-by-field comparison, never source-vs-target hashing.

## Ordered task plan (25 tasks)

> B2a = 1–19, B2b = 20–25. `kind` = pure (unit, no WP) or integration (wp-env). Each task ends in an independently testable deliverable. The `must_handle` items ARE the closed adversarial holes — preserve them as tests.

(The full per-task deliverable / key_interfaces / must_handle / test_focus is the synthesis below; the implementation plans B2a/B2b expand each into TDD steps.)

### B2a — Writers & crosswalk

1. **Schema/crosswalk/state constants** (pure): add `Identifiers::OPTION_TERM_IMAGES_SETTINGS`, `OPTION_MIGRATION_STATE`, `OPTION_PRE_MIGRATION_BACKUP`, `OPTION_MIGRATION_PROGRESS`, `META_DATE_NORMALIZED`; `Crosswalk::LEGACY_TERM_TT_ID`, `LEGACY_COMMENT_ID`, `MIGRATION_COMPLETE`, `LEGACY_SLUG`, `MIGRATION_FLAGS`, `LEGACY_POST_CONTENT`. All back-refs hidden (leading `_`). `LEGACY_POST_CONTENT`/`MIGRATION_FLAGS` hold preserved content/review data → Finalizer allowlist must never strip them.
2. **DateNormalizer** (pure): `normalize(string $raw, ?DateTimeZone $tz=null): ?int` — TZ-anchored `DateTimeImmutable`, not bare `strtotime`; garbage→null, never throws.
3. **PostContentReconciler → 3-arg** (pure): `reconcile($old, ?$desc, ?$temp=null)`; run both `$old` and `$temp` through the safety-net; discard a blob only when visible text is contained AND it carries no shortcodes/structural HTML absent from the description; distinct/ordered backup join; stays pure (square-bracket-token regex, no `has_shortcode`). Existing 5 tests stay green.
4. **TermArtworkMapper** (pure): `remapImages($legacyImages, $ttIdCrosswalk)` → `{images,dropped,conflicts}` with **(int)-cast both sides** of keys; new-tt_id collision → `conflicts` (never silent overwrite). `remapSettings($legacySettings)` → remap taxonomy-name keys via `taxonomyMap()`, **pass through all other (global) keys verbatim**.
5. **LegacyChecksum + Detector refactor** (integration): `LegacyChecksum::forPost(int):string` = exact `md5(post_content . encode(ksort'd raw get_post_meta($id)))`, encoding-hardened; `Detector::sermonChecksum` delegates (its existing tests are the regression guard).
6. **Crosswalk post helpers** (integration): `findNewByLegacyId(legacyId, postType)`, `markLegacy(newId, legacyId)` (`add_post_meta` unique), `migratedPostIds(postType)`, `allMigratedPostIds()` (sermon + podcast). Resolve by **postmeta authoritatively, status-agnostic** (find trashed/auto-draft so re-run doesn't duplicate); `findNewByLegacyId` asserts at-most-one (loud on >1).
7. **Crosswalk term helpers** (integration): `findNewTermByLegacyId(legacyTermId, targetTaxonomy)` **taxonomy-aware**; `markLegacyTerm(newTermId, legacyTermId, legacyTtId)` writes both `LEGACY_TERM_ID` + `LEGACY_TERM_TT_ID`; idempotency probe **bypasses stale term cache** (`$wpdb`/`clean_term_cache`).
8. **TermWriter::migrateTerm** (integration): idempotent via cache-safe probe; `wp_insert_term` into `taxonomyMap()[tax]` copying name/slug/description; on `term_exists` collision with a NATIVE term, create a **new** term with a **deterministic** suffix (`slug.'-legacy-'.id`), flag `slug_collision`, **never** stamp a back-ref onto the native term; assert `!is_wp_error` before `add_term_meta`; record `LEGACY_SLUG`.
9. **TermWriter::migrateAll** (integration): all 5 taxonomies `get_terms(hide_empty=>false)` (orphans migrate); **hard uniqueness guard** — a legacy term_id mapping to >1 new term fails loudly; resumable.
10. **TermCrosswalk reader** (integration): `newTermId(legacyTermId)` (deterministic lowest-id, flag >1) + `ttIdMap()` (legacy tt_id → NEW tt_id from the term's current `term_taxonomy_id`).
11. **ArtworkWriter::migrate** (integration): gate on terms-complete; `remapImages`/`remapSettings`; **add_option-first + backup**; merge-safe (unresolved = pending, not dropped); legacy options untouched.
12. **SermonWriter — post columns + body + back-ref + idempotency** (integration): idempotency gate (`findNewByLegacyId`, complete-vs-partial); preserve all post columns; `post_content` from 3-arg reconciler; **`wp_slash` + KSES-off insert**; back-ref FIRST, `MIGRATION_COMPLETE` LAST; slug-drift flag + `LEGACY_SLUG`; `post_parent` translate-or-zero+flag; attachments referenced (never mutated).
13. **SermonWriter — meta application + date normalization** (integration): write from **per-key unserialized** values; renamed/dropped/unknown-verbatim per mapper; `post_content_temp` verbatim row (single home); multi-value via **delete-then-re-add** (idempotent), single-value replace/unique; emit `sermonator_date_normalized` per **non-numeric row** alongside raw; flags row.
14. **SermonWriter — terms + comments** (integration): assign terms via taxonomy-aware crosswalk (`wp_set_object_terms`, check `WP_Error`, flag missing); re-appliable term repair on existing posts; copy comments depth-first with new ids, remap `comment_parent`, per-comment `LEGACY_COMMENT_ID` (idempotent), copy IP/agent/karma + commentmeta (unserialized); legacy comments untouched.
15. **PodcastWriter + OptionWriter** (integration): podcast mirror reusing SermonWriter disciplines (back-ref FIRST/COMPLETE LAST, KSES-off, unserialized meta, term-filter/default remap); `OptionWriter::migrate` reads `sermonmanager_*` via `OptionMapper`, add_option-first + backup, writes `sermonator_*` + default-podcast.

### B2b — Lifecycle & verification

16. **MigrationState + Orchestrator + batch** (integration): durable per-legacy-id state; sequence terms→artwork→podcasts/options→sermons with **hard preconditions**; single advisory lock (concurrency); chunked/resumable; monotonic state, `verified` only after Verifier passes.
17. **Verifier** (integration): legacy→target completeness (every manifest legacy id has exactly one counterpart with empty failure flags); drift oracle via `LegacyChecksum`; slug-collision compared against `LEGACY_SLUG`; extended drift (terms/options); sets `verified` only when clean.
18. **Rollback** (integration): delete migration-made records in order (posts force=true → terms `wp_delete_term` explicit args → options restore), enumerate comments via `LEGACY_COMMENT_ID`, sweep un-stamped partial-orphan sermonator posts, restore backed-up options; refuse when `finalized`; warn on admin-edited migrated posts; require live legacy source (don't delete a native post with a stray back-ref); post-rollback zero back-refs remain.
19. **Finalizer** (integration): only from `verified` + **fresh drift rescan clean**; per-record delete only for verified counterparts; strip ONLY the pure back-ref allowlist (`LEGACY_POST_ID`/`LEGACY_TERM_ID`/`LEGACY_TERM_TT_ID`/`LEGACY_COMMENT_ID`/`MIGRATION_COMPLETE`), never `LEGACY_POST_CONTENT` or unresolved `MIGRATION_FLAGS`.
20. **WP-CLI** (integration): `wp sermonator detect|migrate|verify|rollback|finalize|status` thin wrappers; destructive commands require `--yes`/confirm + print exact id set; same advisory lock; register only when `defined('WP_CLI')`.
21. **End-to-end full-cycle** (integration): rich `LegacyFixture` (terms w/ desc+slug, artwork seed, comments, serialized meta, temp-only body, non-numeric date) through detect→migrate→verify→rollback(pristine)→re-migrate(no-op)→finalize(gated); before/after proves zero legacy mutation up to Finalize; exercises all 3 gating criteria; crash-injection resume.

## Composer deps

No new production runtime deps — all writers use core WP APIs (`wp_insert_post`, `add_post_meta`, `wp_set_object_terms`, `wp_insert_term`, `get_comments`/`wp_insert_comment`, `get_terms`, add/update/get_option, `$wpdb`). WP-CLI is available in the wp-env `tests-cli` runtime; guard registration with `defined('WP_CLI')`. Existing dev deps (phpunit ^9.6, yoast/wp-test-utils ^1.2) cover unit + wp-env integration. (Action Scheduler evaluated and **not** adopted — a durable-option per-record state machine + chunked WP-Cron/CLI batch is simpler, fully testable, and sufficient for a one-time migration.)
